## 🇬🇧 English

> 🔁 **[点击切换到中文 / Switch to 中文](#-中文)**

Unofficial reimplementation of the paper [**UniSyn: An End-to-End Unified Model for Text-to-Speech and Singing Voice Synthesis**](https://arxiv.org/abs/2212.01546) (Lei et al., AAAI 2023), based on the [VITS](https://github.com/jaywalnut310/vits) repository.

UniSyn is an end-to-end unified speech / singing voice synthesis model that allows you to synthesize **both** the speaking and singing voice of a target speaker, **even when only one of the two data types** (speech-only or singing-only) is available for that speaker.

### ✨ Highlights

- **Unified modeling**: a single model handles both TTS and SVS
- **Unified linguistic features**: text (phoneme + tone) and music score (phoneme + note pitch) share the same input representation
- **MC-VAE**: multi-conditional VAE that splits the latent space into a speaker subspace `z_s` and a residual subspace `z_rst`
- **Guided-VAE**: explicit disentanglement via speaker supervision on `z_s` and pitch supervision on `z_rst`
- **Speaker Timbre Perturbation**: formant perturbation + Wasserstein-distance constraint for stronger speaker-timbre disentanglement
- **End-to-end**: directly generates waveforms from text / score, no separate vocoder needed

### 📐 Architecture

```
           ┌──────────────┐
Text/Score │ Prior Model  │       Speaker ID ──► p(z_s | c_s)
─────────► │ (Text Enc +  │
           │ Length Reg + │──► p(z_rst | c_rst)
           │ Frame Prior) │              │
           └──────────────┘              ▼
                                      ┌─────┐
Audio ──► Posterior Encoder ──► z ──► │ Dec │──► Waveform
                                      └─────┘
                       ▲
                       │
                  KL / GVAE / Perturbation constraints
```

See [Figure 1 of the paper](https://arxiv.org/abs/2212.01546) for the full architecture.

### 📦 Data Preparation

This project trains on two kinds of data:

| Type | Recommended dataset | # Speakers | Duration |
|------|---------------------|------------|----------|
| TTS  | [DataBaker Chinese Female Voice](https://www.data-baker.com/open_source.html) | 1 | ~10h |
| SVS  | [Opencpop](https://wenet.org.cn/opencpop/) | 1 | ~5h |

Directory layout: see *Project Structure* above.

#### 1. Unify sample rate

```bash
python resample.py
```
Resamples every wav under `dataset/` to 24 kHz.

#### 2. Extract log-F0 (for GVAE pitch supervision)

```bash
python preprocess_f0.py
```
Each `.wav` produces a corresponding `.f0.pt`.

#### 3. Formant perturbation (for Speaker Perturbation)

```bash
python preprocess_perturbation.py
```
Each `.wav` produces a `_pert.wav`.

#### 4. Build filelists

```bash
python prepare_filelists.py
```
Produces:
- `filelists/unisyn_train.txt`
- `filelists/unisyn_val.txt`

Each line has the format:
```
<wav_path>|<phone_ids>|<pitch_ids>|<durations>|<pos>|<style_id>|<spk_id>
```
where `style_id`: `0 = TTS`, `1 = SVS`.

### 🚀 Training

```bash
python train.py -c configs/unisyn_base.json -m unisyn_base
```

- Checkpoints are saved under `logs/unisyn_base/`
- TensorBoard:
  ```bash
  tensorboard --logdir logs/unisyn_base
  ```

Main loss terms (see `configs/unisyn_base.json` for details):

| Name | Weight | Description |
|------|--------|-------------|
| `c_mel` | 60.0 | Mel reconstruction loss |
| `c_kl_s` | 12.0 | KL on `z_s` |
| `c_kl_rst` | 1.5 | KL on `z_rst` |
| `c_gvae_s` | 0.0¹ | GVAE speaker supervision |
| `c_gvae_p` | 10.0 | GVAE pitch supervision |
| `c_dur` | 1.5 | Duration prediction L1 |
| `c_fm` | 2.0 | Feature matching |
| `c_adv` | 2.0 | Adversarial loss |
| `c_pert` | 0.02 | Wasserstein perturbation constraint |

¹ For the single-speaker setup we recommend keeping this at 0; raise it to a positive value for multi-speaker training.

### 🎤 Inference

```bash
# TTS: make the SVS speaker talk
python inference.py \
    --checkpoint logs/unisyn_base/G_100000.pth \
    --style 0 \
    --noise_scale 0.3 \
    --length_scale 1.0 \
    --out_prefix tts_demo

# SVS: make the TTS speaker sing
python inference.py \
    --checkpoint logs/unisyn_base/G_100000.pth \
    --style 1 \
    --noise_scale 0.2 \
    --length_scale 1.0 \
    --out_prefix svs_demo \
    --do_recon
```

Arguments:
- `--style`: `0` for TTS, `1` for SVS
- `--noise_scale`: noise amplitude when sampling the latent; larger → more variation, less timbre stability
- `--length_scale`: speed scaling (TTS only; SVS duration is determined by the score)
- `--do_recon`: also run a voice-conversion reconstruction as a sanity check

### 🔍 Differences from VITS

| Aspect | VITS | This project |
|--------|------|--------------|
| Task | TTS only | Unified TTS + SVS |
| Latent space | Single `z` | Split into `z_s` + `z_rst` (MC-VAE) |
| Alignment | Monotonic Alignment Search | Length Regulator + Duration Predictor |
| Prior | Text encoder + Flow | Text Encoder + Frame Prior Network |
| Disentanglement | Implicit through Flow | Explicit GVAE + formant perturbation |
| Inputs | phoneme | phoneme + tone/pitch + dur + pos + style |

### 📝 Deviations from the paper / Known limitations

Because there is no official implementation released, the following details are best-effort reconstructions and may differ from the original paper:

- **Duration input**: the paper treats `dur_note` as a placeholder for TTS; this project feeds `note_dur_input = 0` during TTS training, feeds GT note duration during SVS training, and lets SVS inference rely on the durations given by the score.
- **Pos feature**: recomputed as *relative position within a syllable* (grouped by note for SVS, by toned-final boundary for TTS); not explicitly described in the paper.
- **Prior pitch supervision**: an extra pitch predictor is attached to the FramePriorNetwork output and trained with MSE, to strengthen pitch learning on the prior side (not stated explicitly in the paper).
- **FramePriorNetwork**: a `style_id` embedding is added so the prior network can explicitly distinguish speech vs. singing.
- **Data scale**: the paper uses 2 speakers + 2 singers; we have so far only validated the 1-speaker + 1-singer setup.

## 🇨🇳 中文

> 🔁 **[Click to switch to English / 切换到英文](#-english)**

基于 [VITS](https://github.com/jaywalnut310/vits) 仓库，对论文 [**UniSyn: An End-to-End Unified Model for Text-to-Speech and Singing Voice Synthesis**](https://arxiv.org/abs/2212.01546) (Lei et al., AAAI 2023) 的非官方复刻实现。

UniSyn 是一个端到端的统一语音/歌声合成模型，允许在**只拥有目标说话人的语音数据或歌声数据之一**的情况下，同时合成该说话人的说话和歌唱声音。

### ✨ 主要特性

- **统一建模**：单一模型同时支持 TTS 与 SVS 两种任务
- **统一语言学特征**：文本（phoneme + tone）与乐谱（phoneme + note pitch）共用同一套输入表示
- **MC-VAE**：多条件变分自编码器，将潜空间分解为说话人子空间 `z_s` 与其余信息子空间 `z_rst`
- **Guided-VAE**：通过对 `z_s` 的 speaker 监督和对 `z_rst` 的 pitch 监督强化解耦
- **Speaker Timbre Perturbation**：共振峰扰动 + Wasserstein 距离约束，进一步解耦说话人音色
- **端到端**：直接从文本/乐谱生成波形，无需单独的声码器

### 📐 模型架构

```
           ┌──────────────┐
Text/Score │ Prior Model  │       Speaker ID ──► p(z_s | c_s)
─────────► │ (Text Enc +  │
           │ Length Reg + │──► p(z_rst | c_rst)
           │ Frame Prior) │              │
           └──────────────┘              ▼
                                      ┌─────┐
Audio ──► Posterior Encoder ──► z ──► │ Dec │──► Waveform
                                      └─────┘
                       ▲
                       │
                    KL / GVAE / Perturbation 约束
```

详细结构参见 [论文 Figure 1](https://arxiv.org/abs/2212.01546)。

### 🎤 推理

```bash
# TTS：让 SVS 的说话人开口说话
python inference.py \
    --checkpoint logs/unisyn_base/G_100000.pth \
    --style 0 \
    --noise_scale 0.3 \
    --length_scale 1.0 \
    --out_prefix tts_demo

# SVS：让 TTS 的说话人唱歌
python inference.py \
    --checkpoint logs/unisyn_base/G_100000.pth \
    --style 1 \
    --noise_scale 0.2 \
    --length_scale 1.0 \
    --out_prefix svs_demo \
    --do_recon
```

参数说明：
- `--style`: `0` 代表 TTS，`1` 代表 SVS
- `--noise_scale`: 潜变量采样的噪声幅度，越大生成越多样、音色越不稳定
- `--length_scale`: 语速缩放系数（仅 TTS 生效；SVS 由乐谱决定时长）
- `--do_recon`: 同时跑一次 voice conversion 的重建 sanity check

### 🔍 与 VITS 的主要差异

| 方面 | VITS | 本项目 |
|------|------|--------|
| 任务 | 单任务 TTS | TTS + SVS 统一 |
| 潜空间 | 单一 `z` | 切分为 `z_s` + `z_rst`（MC-VAE） |
| 对齐 | Monotonic Alignment Search | Length Regulator + Duration Predictor |
| 先验 | 文本编码 + Flow | Text Encoder + Frame Prior Network |
| 解耦 | Flow 隐式 | GVAE 显式 + 共振峰扰动 |
| 输入 | phoneme | phoneme + tone/pitch + dur + pos + style |

### 📝 与论文的偏差 / 已知限制

由于论文未开源官方实现，以下细节为复刻推断，可能与原论文有差异：

- **Duration 输入**：论文中 TTS 的 `dur_note` 为占位符；本项目 TTS 训练时 `note_dur_input=0`，SVS 训练时喂入 GT note duration，推理时 SVS 依赖乐谱给定的时长。
- **Pos 特征**：按"音节内相对位置"重新计算（SVS 按音符分组，TTS 按带调韵母边界分组），非论文显式描述。
- **Prior Pitch Supervision**：额外在 FramePriorNetwork 输出端加了一个 pitch predictor 并对其计算 MSE，强化 prior 端音高学习（论文未明说）。
- **FramePriorNetwork**：增加了 `style_id` embedding 输入，让先验网络显式区分说话/唱歌风格。
- **数据规模**：论文使用 2×speaker + 2×singer，本项目目前仅验证了 1 说话人 + 1 歌手的设置。
